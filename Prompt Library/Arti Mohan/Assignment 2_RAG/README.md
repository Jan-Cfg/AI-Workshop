## Objective
The goal is to demonstrate a basic Retrieval-Augmented Generation (RAG) workflow using a simple clean-code knowledge base for a Python project.

## What this prototype does
- loads local text documents
- retrieves relevant documents based on the user question
- compares a plain answer vs a RAG-style grounded answer

## Project Structure
- `app.py` : main Python script
- `docs/` : local text files used as the knowledge source
